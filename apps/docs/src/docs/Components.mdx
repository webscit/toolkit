# Components

All components are added to your project via the CLI:

```bash
npx @webscit/toolkit add <component-name>
```

Browse individual component stories in the sidebar under **Components**. The table below groups all available components by category.

---

## Form inputs

| Component | Description |
|---|---|
| `button` | Clickable action element. Variants: default, destructive, outline, secondary, ghost, link. |
| `button-group` | Arranges related buttons as a visual unit. |
| `checkbox` | Boolean toggle. Built on Base UI Checkbox. |
| `checkbox-group` | Manages a set of related checkboxes with shared state. |
| `combobox` | Searchable single-select input. Combines a text field with a filtered dropdown. |
| `field` | Wrapper that pairs a label, input, and validation message. |
| `input` | Text input. Supports all standard HTML input types. |
| `input-group` | Composes an input with leading/trailing addons (icons, buttons, text). |
| `label` | Accessible label element. Associates with form controls via `for`/`htmlFor`. |
| `native-select` | Browser-native `<select>` element with toolkit styling. |
| `radio` | Single radio option. Use inside `radio-group`. |
| `radio-group` | Manages a set of mutually exclusive radio options. |
| `select` | Custom single-select dropdown. Built on Base UI Select. |
| `slider` | Range input for numeric values. |
| `switch` | Toggle between two states. Built on Base UI Switch. |
| `textarea` | Multi-line text input. |
| `toggle` | Pressable button with on/off state. |
| `toggle-group` | Group of related toggles with single or multiple selection. |

---

## Layout

| Component | Description |
|---|---|
| `accordion` | Collapsible content sections, one open at a time. |
| `aspect-ratio` | Constrains a child element to a fixed aspect ratio. |
| `card` | Surface for grouping related content. Composed of card, header, title, description, content, and footer. |
| `collapsible` | A single show/hide container. |
| `direction` | Sets text direction context (`ltr`/`rtl`) for a subtree. |
| `resizable` | Draggable split-panel layout. |
| `scroll-area` | Custom-styled scrollable container. |
| `separator` | Horizontal or vertical dividing line. |
| `sidebar` | Application sidebar with collapsible navigation sections. |
| `table` | Semantic HTML table with toolkit styling. |

---

## Feedback

| Component | Description |
|---|---|
| `alert` | Non-interactive message for informational, warning, or error states. |
| `badge` | Compact label for status or category. |
| `empty` | Placeholder for empty states. |
| `kbd` | Keyboard shortcut display. |
| `progress` | Linear progress indicator. |
| `skeleton` | Loading placeholder that matches the shape of expected content. |
| `spinner` | Circular indeterminate loading indicator. |
| `toast` | Transient notification message. |

---

## Navigation

| Component | Description |
|---|---|
| `breadcrumb` | Shows the current location in a page hierarchy. |
| `navigation-menu` | Horizontal navigation with dropdown panels. |
| `pagination` | Page navigation controls for paginated data. |
| `tabs` | Tabbed content panels. Built on Base UI Tabs. |

---

## Overlay and dialog

| Component | Description |
|---|---|
| `alert-dialog` | Modal confirmation dialog with destructive action pattern. |
| `command` | Command palette with keyboard-driven search and selection. |
| `context-menu` | Right-click context menu. |
| `dialog` | Modal dialog. Built on Base UI Dialog. |
| `drawer` | Panel that slides in from a screen edge. |
| `hover-card` | Rich tooltip-like card that appears on hover. |
| `menu` | Dropdown menu. Built on Base UI Menu. |
| `menubar` | Horizontal menu bar with nested dropdowns. |
| `popover` | Non-modal floating panel anchored to a trigger. |
| `sheet` | Full-height side panel (a specialised drawer). |
| `tooltip` | Short label that appears on hover. Built on Base UI Tooltip. |

---

## Adding a component

```bash
# Single component
npx @webscit/toolkit add dialog

# Multiple components at once
npx @webscit/toolkit add dialog button label
```

After adding, import tokens if you have not already done so. See [Installation](?path=/docs/docs-installation--docs) for the full setup steps.
